The Problem

  • Your Azure environment generates signals everywhere
  • No single tool sees the complete picture
  • Security reviews are manual, fragmented, and slow
  • Built-in Azure Policies cover common scenarios — but your organization has unique requirements
  • The gaps between tools are where the risks hide
"Your Azure environment is talking — but who's listening?"

Each tool sees a piece. None sees the whole picture.

Tool #1

Maester

Open-source PowerShell framework for Entra ID security testing

Built on Pester · Tests via Microsoft Graph API · CI/CD ready

  • 40+ pre-built tests from Entra ID Security Config Analyzer (EIDSCA)
  • Conditional Access "What-If" simulation
  • Aligned to MITRE ATT&CK for Entra ID
  • Interactive HTML reports with pass/fail and remediation links
  • GitHub Actions & Azure DevOps integration

Tool #2

Az Quick Review (azqr)

CLI tool scanning Azure resources against Microsoft best practices

70+ resource types · Resource Graph + Azure Go SDK · Excel/CSV/JSON output

  • Aligned to Azure Proactive Resiliency Library v2
  • Covers reliability, operational excellence, cost optimization
  • Optionally pulls Advisor, Defender for Cloud, and Policy compliance data
  • Multi-sheet Excel workbooks with per-resource recommendations

Tool #3

Azure Review Checklists

Version-controlled checklists for validating Azure designs

Landing Zone · Well-Architected · AKS · AVD · Cost Optimization · and more

  • Mix of manual review items and Azure Resource Graph queries
  • The "architectural design review" layer
  • Catches design-level gaps that resource scanners miss

The Gap

What none of these tools do:

  • They detect issues — but they don't prevent them
  • Azure Policies are the enforcement layer
  • But built-in policies don't cover everything
  • Your organization has requirements Microsoft hasn't built policies for
This is where AI comes in.

Real-World Example

MySQL Flexible Server Guardrails

The problem: Built-in policies cover the basics, but miss critical configurations.

What built-in covers:

  • Advanced Threat Protection
  • Infrastructure encryption

What was missing:

  • Minimum version enforcement
  • SSL/TLS enforcement
  • Slow query logging
  • PII-safe log output
  • Customer-managed keys
  • Geo-redundant backup

Key Takeaways

1. No single tool gives you the full picture — layer your security review
2. AI is the cross-reference engine — it connects dots across tool outputs
3. Custom Azure Policies fill the gaps — AI can generate them from docs and real issues
4. Iterate and validate — AI gets you 90% there, always verify aliases and schemas
5. Enforce progressively — audit first, deny in production
